# Desafio de Prática Git e GitHub

Este repositório foi criado para consolidar os conceitos de controle de versão, colaboração em equipe e formatação de documentos ultilizado Markdown, através da plataforma DIO.

## Tecnologias e Ferramentas
* **Git** - Controle de versão local.
* **GitHub** - Hospedagem de repositórios e colaboração.
* **GitHub.dev** - Editor de código baseado na nuvem.
* **Markdown** - Formatação de texto.

## Objetivos Concluídos
1. Criação do repositório público com README.
2. Uso do editor web (GitHub.dev).
3. Prática de commitd e controle de alterações.
4. Fluxo de colaboração (Issues e Pull Requests).

---
*Projeto desenvolvido para fins de estedo técnico.*

## 🤝 Como Contribuir
1. Faça um **Fork** do projeto.
2. Crie uma nova **Branch** para sua modificação (`git checkout -b feature/nova-mudanca`).
3. Faça o **Commit** de suas alterações (`git commit -m 'feat: adiciona nova mudanca'`).
4. Abra um **Pull Request** para avaliação.
